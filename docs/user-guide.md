# คู่มือเริ่มต้นใช้งาน CropWat Auto-runner

คู่มือนี้เขียนสำหรับผู้ใช้ทั่วไป (ไม่ต้องเป็นโปรแกรมเมอร์) — ถ้าอยากรู้รายละเอียด
ทางเทคนิค/โครงสร้างโปรแกรม ดูที่ `README.md` แทน

## สิ่งที่ต้องมีก่อนเริ่ม

- Windows (โปรแกรมนี้ทำงานบน Windows เท่านั้น เพราะควบคุม CropWat ผ่านกลไกของ
  Windows โดยตรง)
- ติดตั้ง [CropWat 8.0](https://www.fao.org/land-water/databases-and-software/cropwat/en/)
  ไว้แล้ว (โปรแกรมของ FAO ตัวจริง — CropWat Auto-runner แค่ควบคุมมันให้ ไม่ได้
  แทนที่มัน)
- ข้อมูล climate/rain/crop/soil ที่จะใช้รัน (ดูกติกาตั้งชื่อไฟล์ด้านล่าง)

## 1. ดาวน์โหลดและเปิดโปรแกรม

1. ไปที่หน้า [Releases](../../releases) แล้วดาวน์โหลด `CropWatAutoRunner.exe`
   เวอร์ชันล่าสุด (ไฟล์เดียว ไม่ต้องติดตั้ง)
2. เปิดไฟล์ — ถ้า Windows ขึ้นเตือน **"Windows protected your PC"**
   (SmartScreen) ให้กด **More info** แล้วกด **Run anyway** — ขึ้นแบบนี้เพราะไฟล์
   ไม่ได้ซื้อใบรับรองลงนามโค้ด (code signing) ไม่ได้แปลว่ามีไวรัส
3. โปรแกรมจะเปิดหน้าต่างเบราว์เซอร์ขึ้นมาเอง (เป็นหน้าจอของโปรแกรม ทำงานบน
   เครื่องเราเอง ไม่ได้ส่งข้อมูลออกไปไหน)

## 2. เตรียมข้อมูล — กติกาตั้งชื่อไฟล์ (สำคัญมาก)

ระบบอ่าน "ปี + เดือน" จากชื่อไฟล์โดยตรง **ชื่อไม่ตรงกติกา = สแกนไม่เจอ** (ตัวใหญ่/
เล็กไม่มีผล จะซ้อนโฟลเดอร์ย่อยกี่ชั้นก็ได้ ไม่ต้องเรียงเหมือนกันทุกสถานี):

| ประเภท | กติกา | ตัวอย่าง |
|---|---|---|
| โฟลเดอร์สถานี | ขึ้นต้น `Clim_` หรือ `Rain_` | `Clim_425201`, `Rain_450006` |
| ไฟล์ climate | `{รหัสสถานี}clim_{ปี}{เดือน3ตัว}.PED` | `425201clim_1981apr.PED` |
| ไฟล์ rain | `rain_{รหัสสถานี}_{ปี}{เดือน3ตัว}.CRD` | `Rain_450006_1981Apr.CRD` |
| ไฟล์พืช/ดิน | นามสกุล `.CRO` / `.SOI` ชื่ออะไรก็ได้ | `MAIZE.CRO`, `SANDYLOAM.SOI` |

รวมทุกอย่าง (สถานี + พืช + ดิน) ไว้ใน **โฟลเดอร์เดียว** — โปรแกรมสแกนลึกเข้าไปเอง

## 3. ตั้งค่าครั้งแรก (แท็บ "ตั้งค่า")

1. วาง path โฟลเดอร์ข้อมูลต้นทาง (ข้อ 2) ที่ "โฟลเดอร์ข้อมูลต้นทาง" แล้วกด
   **สแกน** — ตรวจว่าเจอสถานี/ไฟล์ครบตามที่คาด
2. วาง path โฟลเดอร์ที่จะเก็บผลลัพธ์ที่ "โฟลเดอร์ผลลัพธ์"
3. กด **บันทึกการตั้งค่า** — ทำครั้งเดียว ค่าจะอยู่ถาวรแม้ปิดโปรแกรมไปแล้วเปิดใหม่

## 4. เปิด CropWat 8.0 ทิ้งไว้

เปิดโปรแกรม CropWat 8.0 เปล่าๆ พอ **ไม่ต้องเปิดไฟล์อะไรเอง** — CropWat
Auto-runner จะเปิดไฟล์ climate/rain/crop/soil ให้เองทั้งหมดตามแผนที่ตั้งไว้

เปิดเร็วขึ้นได้: ตั้งค่า path ไฟล์ `.exe` ของ CropWat ไว้ที่การ์ด "โปรแกรม
CropWat 8.0" ในแท็บตั้งค่า (กด "เลือกไฟล์" เลือกได้เลย) แล้วจะมีปุ่ม **"เปิด
CropWat"** ที่แท็บ Dashboard ให้กดเปิดโปรแกรมได้ทันที ไม่ต้องไปหาไอคอนเอง

## 5. เลือกวันปลูกที่จะทดลอง + เริ่มรัน (แท็บ "Dashboard")

1. เลือกช่วงปีและวันปลูกที่ต้องการทดลองในปฏิทิน (ปฏิทินบันทึกอัตโนมัติ)
2. กด **เริ่มรันทั้งหมด** (หรือกด `Ctrl+Alt+F9` จากที่ไหนก็ได้ในเครื่อง)
3. ก่อนเริ่มจริง ระบบจะ**สแกนตรวจไฟล์ climate/rain ให้ครบทุกวันปลูกที่วางแผนไว้
   ก่อนเสมอ** — ถ้าเจอวันปลูกที่ไม่มีไฟล์ข้อมูลย้อนหลังให้ใช้ (เช่น สถานีเริ่มมี
   ข้อมูลทีหลังเดือนที่เลือก) จะขึ้นกล่องแจ้งเตือนรายละเอียดให้ก่อน ต้องกด
   ยืนยันเองถึงจะเริ่มรันได้ (วันปลูกที่มีปัญหาจะขึ้นสถานะ "มีปัญหา" ส่วนวันปลูก
   อื่นที่มีไฟล์ครบจะรันตามปกติ)
4. **โหมดเบื้องหลัง** (ค่าเริ่มต้น) ใช้คอมทำงานอื่นระหว่างรันได้เลย แค่**อย่าย่อ
   (minimize) หน้าต่าง CropWat** — วางไว้หลังหน้าต่างอื่นได้ปกติ
5. ดูความคืบหน้าได้จาก: แถบลอยมุมขวาล่างจอ, ไอคอนถาดงาน (system tray), หรือ
   หน้าเว็บของโปรแกรมเอง — `Ctrl+Alt+F10` = หยุดรัน

## 6. สร้างไฟล์ผลลัพธ์

หลังรันเสร็จ (หรือรันไปบางส่วนแล้วอยากดูผลระหว่างทาง) กด:
- **สร้าง/อัปเดต Excel** — รวมผลทุกวันปลูกที่รันสำเร็จเป็นตาราง Result เดียว
- **สร้างไฟล์ Word (ภาพ)** — รวม screenshot ของแต่ละวันปลูกเป็นเอกสารเดียว

ทั้งสองปุ่มกดซ้ำได้ทุกเมื่อ สร้างใหม่จากข้อมูล `.txt`/screenshot ที่มีอยู่จริง
ในโฟลเดอร์ผลลัพธ์เสมอ (ไม่ต้องรันใหม่ทั้งหมดถ้าไฟล์เดิมยังอยู่)

## ปัญหาที่พบบ่อย

**เปิดโปรแกรมแล้วค้างที่ "กำลังเชื่อมต่อ..." ไม่ขยับ**
มักเกิดหลังอัปเดตเวอร์ชัน — ลองปิดหน้าต่างเบราว์เซอร์ (อย่าปิดแค่ tab ถ้าเปิดค้าง
จากเวอร์ชันเก่า) แล้วเปิดโปรแกรมใหม่อีกครั้ง ตั้งแต่ v0.5.22 เป็นต้นไปเบราว์เซอร์
จะไม่ cache หน้าเก่าค้างแบบนี้แล้ว

**กด "เริ่มรันทั้งหมด" แล้วขึ้นกล่องเตือนว่าบางวันปลูก "ไม่มีไฟล์"**
แปลว่าสถานี climate/rain ที่เลือกไว้มีข้อมูลเริ่มทีหลังเดือนที่ต้องการทดลองปลูก
จริง (เช่น อยากทดลองปลูกกุมภาพันธ์ แต่สถานีมีข้อมูลเริ่มเมษายนเป็นต้นไป) ระบบจะ
ไม่เดาใช้ข้อมูลเดือนอื่นแทนให้เองเด็ดขาด (จะทำให้ผลเพี้ยน) ต้องเลือกยืนยันเองว่า
จะข้ามวันปลูกเหล่านั้นไปหรือเปลี่ยนวันปลูกที่จะทดลอง

**คลิกไปโดนหน้าต่าง CropWat ระหว่างรันเบื้องหลัง แล้วเกิดปัญหา**
ตั้งแต่ v0.5.18 มีหน้าต่างโปร่งใส "shield" กันคลิกพลาดอัตโนมัติเฉพาะตอนที่
CropWat เป็นหน้าต่างที่ active อยู่ ถ้ายังเจอปัญหาแนะนำปิด CropWat แล้วเปิดใหม่
ก่อนรันรอบถัดไป (การคลิกพลาดระหว่างรันอาจทำให้ CropWat ค้างชั่วคราว)

**CropWat ค้างสนิท กดปิดเองไม่ได้เลย (ต้องเข้า Task Manager)**
บางครั้งระหว่างรันเบื้องหลังอาจมี dialog ที่ระบบไม่รู้จักโผล่ขึ้นมาแล้วโดนซ่อนไป
พร้อมกับหน้าต่างอื่นๆ ที่รู้จัก ทำให้ CropWat ค้างสนิทมองไม่เห็น dialog นั้นเลย —
ตั้งแต่ v0.5.25 มีปุ่ม **"CropWat ค้าง? ปิดฉุกเฉิน"** ในแท็บ Dashboard (ข้างปุ่ม
เริ่มรัน/หยุด) กดแล้วจะบังคับปิด CropWat ทันที + รีเซ็ตสถานะโปรแกรมนี้กลับมา
พร้อมเริ่มรันใหม่ได้เลย **ไม่ต้องปิด-เปิดโปรแกรมนี้ทั้งตัวอีกต่อไป** — ไฟล์ .txt
ของวันปลูกที่ทำเสร็จไปแล้วจะไม่หายไปไหน แค่เปิด CropWat ขึ้นมาใหม่แล้วกดเริ่มรัน
ต่อได้ทันที (ระบบมีกลไกพยายามดึงหน้าต่างที่ค้างกลับมาให้เห็น/คลิกได้เองก่อนถึง
จุดที่ต้องกดปุ่มนี้ด้วย แต่ถ้าไม่ทันหรือไม่ได้ผล ปุ่มนี้คือทางออกสุดท้ายที่ชัวร์
100%)

**วันปลูกล้มเหลวหมดทั้งปี ข้อความ error พูดถึงหน้าต่างโมดูลซ้ำ (TDayEToPMForm/
Duplicate window)**
มักเกิดจากมีหน้าต่างโมดูล Climate/Rain ค้างอยู่จากรอบก่อนหน้า (เช่นรันค้างไว้
กลางทางแล้วปิดโปรแกรมไม่สะอาด) แล้วระบบปิดหน้าต่างเก่าเหล่านั้นไม่สำเร็จ **วิธี
แก้ที่ชัวร์ที่สุดตอนนี้**: ปิด CropWat ให้สนิท (ใช้ปุ่ม "CropWat ค้าง? ปิด
ฉุกเฉิน" ถ้ากด X ธรรมดาไม่ติด) แล้วเปิด CropWat ขึ้นมาใหม่เปล่าๆ ก่อนเริ่มรัน
รอบถัดไปเสมอ — ไม่ต้องเปิดไฟล์อะไรเอง ระบบเปิดให้เองทั้งหมดตามปกติ (ปัญหานี้
กำลังปรับปรุงอยู่เรื่อยๆ ถ้ายังเจอบ่อยรบกวนแจ้งรายละเอียด/screenshot มาได้เลย)

**อยากรันแบบชัวร์ 100% ไม่ให้กระทบเครื่องจริงเลย**
ดู `docs/vm-guide.md` — วิธีรันใน Virtual Machine แยกขาดจากเครื่องจริง

## อัปเดตโปรแกรม

โปรแกรมเช็คเวอร์ชันใหม่ให้อัตโนมัติ ถ้ามีเวอร์ชันใหม่จะขึ้นแจ้งเตือนในตัวโปรแกรม
กดอัปเดตได้จากในนั้นเลย (ต้องต่ออินเทอร์เน็ต)
