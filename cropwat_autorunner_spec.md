# CropWat Auto-runner — Design Spec

## บริบทงาน
ผู้ใช้รัน CropWat (โปรแกรม Windows desktop) เพื่อคำนวณ Irrigation Schedule ทีละปี
ตั้งแต่ปี 1981–2025 (45 ปี) แล้ว copy ผลลัพธ์ไปรวมใน Excel master ทีละปี
ต้องการระบบที่ทำขั้นตอนนี้อัตโนมัติทั้งหมด

## สถาปัตยกรรม
```
เว็บหน้าจอ (browser, localhost)
        |
Backend (Python, รันบนเครื่องเดียวกับ CropWat)
   - Automation engine (pywinauto) -> ควบคุม CropWat
   - File engine -> อ่าน/เขียนไฟล์
        |
   +----+----------------+
   |                      |
CropWat 8.0          ไฟล์ข้อมูล
(desktop app)        - climate/rain (.CLI/.PEN) ต่อปี
                      - export ผลลัพธ์ (CSV/txt) ต่อปี
                      - Excel master (รวมทุกปี)
                      - list วันปลูกต่อปี (Excel ที่มีอยู่แล้ว)
```

หมายเหตุสำคัญ: ต้องเป็น **local web app** เท่านั้น (backend รันบนเครื่อง
ผู้ใช้เอง ไม่ใช่ cloud) เพราะต้องมีสิทธิ์ควบคุมโปรแกรม desktop โดยตรง —
เบราว์เซอร์ปกติทำแบบนี้ไม่ได้

## ข้อมูล input ที่มีอยู่แล้ว
- ไฟล์ climate/rain แบบ .CLI/.PEN ของ CropWat (ต่อปี/สถานี)
- List วันปลูกพืชแต่ละปี (Excel)
- crop/soil ไฟล์คงที่ ไม่เปลี่ยนตามปี

## โครงสร้างโฟลเดอร์ไฟล์ climate/rain (shift year)

ไฟล์ climate (.CLI) และ rain (.PEN) **ไม่ได้เก็บแบบเริ่มมกราคมเสมอไป**
แต่ละไฟล์เป็นชุดข้อมูลต่อเนื่อง 12 เดือน ที่เริ่มจากเดือนไหนก็ได้ เพื่อให้
ฤดูปลูกที่คร่อมปี (เช่น ปลูก พ.ย. 2024 → เก็บเกี่ยว มี.ค. 2025) มีข้อมูล
ภูมิอากาศครบทั้งช่วง ไม่ขาดตอนตรงรอยต่อปี

โครงสร้างจริง (ตัวอย่าง):
```
cropwat/
  Clim_425201/                (climate ต่อสถานี)
    Clim_425201/
      Clim_425201/
        01_Jan/  02_Feb/  03_Mar/  04_Apr/  05_May/
        06_Jun/  07_Jul/  08_Aug/  Nov/     Dec/
  Rain_450006/                (rain ต่อสถานี)
    04_Rain_450006Apr/
    05_Rain_450006May/
    06_Rain_450006Jun/
    07_Rain_450006Jul/
    08_Rain_450006Aug/
  MAIZE_120/                  (crop ไฟล์ คงที่)
  sandy loam/                 (soil ไฟล์ คงที่)
```
หมายเหตุ: ชื่อโฟลเดอร์ไม่ consistent 100% — บางอันมีเลขเดือนนำหน้า
(`04_Apr`) บางอันไม่มี (`Nov`, `Dec`) ต้อง parse ชื่อเดือนแบบยืดหยุ่น
ไม่ใช่ hardcode รูปแบบเดียว

**สำคัญ**: ไม่รู้ล่วงหน้าว่าสถานี/พืชหนึ่งๆ จะมีโฟลเดอร์เดือนเริ่มต้นครบ
1–12 หรือไม่ (แต่ละสถานี/พืชมีไม่เท่ากัน) — backend ต้อง**สแกนโฟลเดอร์จริง
ที่มีอยู่**ก่อนเลือกไฟล์ ห้าม hardcode รายชื่อเดือนที่มีไว้ล่วงหน้า

### รูปแบบชื่อไฟล์จริง (ยืนยันจากไฟล์จริงในเครื่องผู้ใช้)

**นามสกุลไฟล์จริงคือ `.CRD` (rain) และ `.PED` (climate)** — ไม่ใช่
.CLI/.PEN ตามที่เข้าใจไว้ตอนแรก ต้องแก้ไขทุกจุดใน implementation ที่
อ้างอิงนามสกุลไฟล์

ในแต่ละโฟลเดอร์เดือนเริ่มต้น (เช่น `06_Jun`) จะมีไฟล์ **หนึ่งไฟล์ต่อหนึ่งปี**
เรียงตั้งแต่ 1981 เป็นต้นไป ชื่อไฟล์ตามรูปแบบ:

- **Rain**: `Rain_{rain_station_id}_{year}{MonAbbrev}.CRD`
  ตัวอย่าง: `Rain_450006_1981Jun.CRD`
  - เดือนย่อขึ้นต้นด้วยตัวใหญ่ (`Jun`, `Apr`, `Nov`, `Dec`)
- **Climate**: `{clim_station_id}clim_{year}{monabbrev}.PED`
  ตัวอย่าง: `425201clim_1981jun.PED`
  - เดือนย่อเป็นตัวเล็กทั้งหมด (`jun`, `apr`, `nov`, `dec`)

**ข้อควรระวังสำหรับ implementation**: ตัวพิมพ์ใหญ่/เล็กของเดือนไม่ตรงกัน
ระหว่าง rain กับ climate (`Jun` vs `jun`) ห้าม hardcode case เดียวใช้ทั้ง
สองฝั่ง ต้องแยก formatter ของแต่ละประเภทไฟล์ชัดเจน และควรใช้วิธี
**list ไฟล์จริงในโฟลเดอร์แล้ว match ด้วย regex/pattern แบบ
case-insensitive** แทนการ generate ชื่อไฟล์ขึ้นมาเองแล้วเดาว่าตรง
เพราะเสี่ยง fail ถ้ามีความไม่ consistent อื่นที่ยังไม่เจอ

### กติกาเลือกโฟลเดอร์ตามเดือนปลูก
ให้ `planting_month` = เดือนปลูกของปีนั้น (จาก list วันปลูกที่เตรียมไว้)

1. หาโฟลเดอร์เดือนเริ่มต้นทั้งหมดที่มีอยู่จริงสำหรับสถานี/พืชนั้น
2. เลือกโฟลเดอร์ที่ **เดือนเริ่มต้น ≤ planting_month และมากที่สุด**
   ในบรรดาที่ไม่เกิน (คือใกล้เคียงที่สุดแบบไม่เกินเดือนปลูก)
   - เช่น ปลูกเดือน 7 (ก.ค.), มีตัวเลือก 04 กับ 06 → เลือก **06**
     (ใกล้กว่าและไม่เกิน)
3. ถ้าไม่มีโฟลเดอร์ไหนเลยที่เดือนเริ่มต้น ≤ planting_month (เช่น ปลูก
   เดือน 3 แต่มีแค่โฟลเดอร์เริ่ม 04 กับ 06) ให้ **ข้ามไปหาโฟลเดอร์ของปีก่อนหน้า**
   ที่เดือนเริ่มต้นมากที่สุด (มักจะเป็นเดือนท้ายปีอย่าง 11 หรือ 12)
4. Logic นี้ใช้เหมือนกันทั้ง climate folder และ rain folder (เลือกแยกกัน
   ได้ กรณีสถานี climate กับสถานี rain มีเดือนเริ่มต้นให้เลือกไม่ตรงกัน)

## หน้าจอ Options ของ CropWat (global settings)
พบว่า CropWat มีหน้าต่าง "CROPWAT options" แยกจากหน้าเปิดไฟล์/รันคำนวณ
ปกติ มีแท็บ: Climate/ETo, Rainfall, Non-rice crop scheduling, Rice
scheduling, Land Preparation (rice) — ตั้งค่าเช่น สูตร ETo, สูตร
effective rainfall, irrigation timing/efficiency

**สมมติฐานที่ต้อง confirm กับ user ก่อนเขียน automation**: ค่าพวกนี้
เป็น global setting ที่ตั้งครั้งเดียวแล้วคงที่ทุกปี (ไม่ต้องไปยุ่งทุกรอบ
ที่ automate) ใช่หรือไม่ ถ้าใช่ ให้ automation engine **ข้ามหน้านี้ไปเลย**
ไม่ต้องคลิกทุกรอบ — เข้าไปตั้งครั้งเดียวตอน setup แล้วกด "Save as
default" ก็พอ

## Flow การรันต่อ 1 ปี
1. เปิดไฟล์ climate/rain ของปีนั้นใน CropWat
2. ตั้งวันปลูกใน module Crop ตามค่าจาก list ที่เตรียมไว้
3. สั่งคำนวณ (calculate)
4. เปิดหน้า Irrigation Schedule
5. กด Export/Print -> ได้ไฟล์ CSV/txt
6. Backend อ่านไฟล์ CSV/txt แล้ว merge เข้า Excel master
   (โครงสร้างตรงตาม sheet "Result" ที่มีอยู่: ต่อปีมี 5 แถว
   — % yield reduction, Actual water use, Potential water use,
   Total rainfall, Effective rainfall — คอลัมน์เป็นวันในฤดูปลูก)
7. บันทึกสถานะปีนั้น (สำเร็จ / ล้มเหลว / ต้องตรวจสอบ) แล้วไปปีถัดไป

## กรณี edge case ที่ต้องรองรับ
พบจากไฟล์ตัวอย่างจริงของผู้ใช้ (sheet "หมายเหตุ" / คอลัมน์ note ใน
Result-1): บางปีข้อมูล climate ไม่ครบ (เช่น "ขาดข้อมูล Sun เดือน 4",
"ขาดข้อมูล Sun เดือน 12") ทำให้ CropWat รันได้ไม่ครบทั้งปี หรือรันไม่ได้เลย
ระบบต้อง:
- ตรวจจับ error/ผลลัพธ์ผิดปกติจาก CropWat ระหว่างรัน (ไม่ใช่แค่ assume ว่าสำเร็จ)
- บันทึกปีที่มีปัญหาไว้ในสถานะ "ต้องตรวจสอบ" พร้อมข้อความ error ถ้ามี
- ไม่ทำให้ระบบหยุดทั้งหมดเมื่อ 1 ปี error — ต้องข้ามไปปีถัดไปต่อได้
- มีปุ่มให้ "รันปีที่ค้าง/error ใหม่" หลังแก้ไขข้อมูลแล้ว

## หน้าจอ (ดู mockup ที่แนบมาในแชท)
### Dashboard หลัก
- ตั้งช่วงปีเริ่มต้น–สิ้นสุด (default 1981–2025)
- ปุ่ม "เริ่มรันทั้งหมด"
- Progress bar + ตัวนับ (เช่น 12/45 ปี) + ปีที่กำลังรันอยู่
- รายการสถานะรายปี แสดง 4 สถานะ: เสร็จ / กำลังรัน / รอคิว / มีปัญหา
  (มีปัญหา ต้องโชว์ error message สั้นๆ ด้วย)
- ปุ่ม "ดาวน์โหลด Excel รวม"
- ปุ่ม "รันปีที่ค้างใหม่" (รันเฉพาะปีที่สถานะ "มีปัญหา")

### หน้าตั้งค่า (ต้องออกแบบเพิ่ม — ยังไม่ได้ทำ mockup)
ควรมี: path โฟลเดอร์ไฟล์ climate/rain, path ไฟล์ list วันปลูก,
path ไฟล์ crop/soil, path Excel master ปลายทาง

## สิ่งที่ backend ต้องรู้จาก CropWat (ยังไม่ทราบ — รอ user สแกนให้)
ชื่อ control/เมนูจริงในหน้าต่างต่อไปนี้ (ใช้ pywinauto สแกนด้วยสคริปต์
inspect_cropwat.py ที่ให้ไปก่อนหน้านี้):
- หน้าเปิดไฟล์ Climate/Rain
- หน้า Crop (ช่องตั้งวันปลูก)
- หน้า Irrigation Schedule + ปุ่ม Export/Print
- รูปแบบไฟล์ที่ export ออกมา (คอลัมน์/หัวตาราง จริง)

## ตัวอย่างไฟล์ผลลัพธ์ที่ต้องได้ (structure ปลายทาง)
ผู้ใช้แนบไฟล์ตัวอย่างจริงมาแล้ว: `Maize_C425201_R450006SandLo_231068.xlsx`
- sheet "Result": ต่อปี 5 แถว (ตัวแปรที่ระบุด้านบน) x คอลัมน์วันในฤดูปลูก
- sheet "Result-1": อีกมุมมองหนึ่ง (ENSO pattern ต่อปี + ค่ารายวัน)
- sheet "หมายเหตุ": metadata (station, soil, crop, บันทึกปัญหารายปี)
